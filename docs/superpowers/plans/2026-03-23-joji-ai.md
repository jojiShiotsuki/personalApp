# Joji AI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a personal AI assistant inside the Vertex CRM that uses an Obsidian vault (synced via GitHub) as its knowledge base, with conversation persistence, CRM tool use, and both a floating chat panel and full-page `/ai` interface.

**Architecture:** Three layers -- Knowledge (vault sync + embeddings), AI (Claude Sonnet/Opus with tools), Interface (floating panel + full page). Vault syncs from GitHub, gets chunked and embedded via Voyage AI, searched via numpy cosine similarity. Conversations persist in the database. SSE streaming for real-time responses.

**Tech Stack:** FastAPI, SQLAlchemy, SQLite/PostgreSQL, Alembic, Voyage AI, GitPython, numpy, React 18, TypeScript, TanStack Query, TailwindCSS

**Spec:** `docs/superpowers/specs/2026-03-23-joji-ai-design.md`

### Task 5: Implement VaultSyncService

**Files:**
- Create: `backend/app/services/vault_sync_service.py`

- [ ] **Step 1: Implement the sync service**

Create `VaultSyncService` class with:

- `__init__()` -- initializes Voyage AI client with `VOYAGE_API_KEY`
- `sync_vault(db, settings)` -- main flow:
  1. Decrypt GitHub token from settings
  2. Clone repo to `backend/data/vault-repo/` (first time) or `git pull` (subsequent)
  3. Walk all `.md` files (skip `crm-sync/` for now)
  4. Hash each file, compare to VaultFile records
  5. For changed/new files: delete old chunks, create new ones
  6. For deleted files: remove VaultFile + chunks
  7. Batch: commit after every 20 files to avoid long DB locks
  8. Update settings.last_sync_at, last_sync_status, last_sync_file_count
- `chunk_markdown(content, file_path) -> list[dict]` -- splits into ~500 token chunks:
  - Split on headings (##, ###, etc.) -- heading always starts new chunk
  - Keep code blocks intact even if over limit
  - Extract frontmatter as metadata (not chunked)
  - Track heading hierarchy per chunk (e.g. "SOPs > Pricing > Hourly Rates")
  - Preserve `[[wikilinks]]` in chunk text
- `embed_chunks(texts: list[str]) -> list[list[float]]` -- calls Voyage AI `voyage-3-lite` model, returns 512-dim embeddings
- Error handling: GitHub unreachable -> log + keep existing index. Voyage AI down -> mark sync "partial". Single file fails -> skip, continue.

- [ ] **Step 2: Create data directory**

Run: `mkdir -p backend/data` and add `backend/data/` to `.gitignore`

- `search(db, query, top_k=5) -> list[dict]` -- embed query via Voyage AI, compute cosine similarity against cache, return top_k results with content, file_path, heading_context, chunk_id

- Fallback: if Voyage AI is down, use `VaultChunk.content LIKE '%query%'` with word splitting

- Max tokens: 4096
- New `chat_stream(db, user_id, message, conversation_id, model_override)` method:
  1. Load or create Conversation record
  2. Save user message as ConversationMessage
  3. Search vault for relevant chunks (top 5) using VaultSearchService
  4. Build system prompt with vault context block (from spec)

  6. Call Claude with tools, stream response
  7. Yield SSE events: `event: text`, `event: tool_call`, `event: tool_result`, `event: vault_ref`, `event: error`, `event: done`
  8. On completion: save assistant message as ConversationMessage with model, tokens_used, cost_usd, vault_chunks_used
- System prompt built dynamically: checks for `sops/about-me.md` vault chunk, falls back to default
- Keep the existing `_build_system_prompt` for backward compat but add vault context injection

- [ ] **Step 1: Create the routes file**

Create `backend/app/routes/joji_ai.py` with all endpoints. All routes require `get_current_user` dependency. Follow patterns from `cold_outreach.py`:

**Chat endpoint:**
- `POST /api/ai/chat` -- accepts ChatRequest body (message, conversation_id, model). Returns StreamingResponse with SSE. Creates conversation if none specified. Uses JojiAIService.chat_stream().

**Conversation endpoints:**
- `GET /api/ai/conversations` -- list user's conversations, paginated (limit/offset), ordered by updated_at desc. Returns list of ConversationResponse.
- `GET /api/ai/conversations/{id}` -- get single conversation with messages (paginated). Returns ConversationWithMessages.
- `DELETE /api/ai/conversations/{id}` -- delete conversation + cascade messages.
- `POST /api/ai/conversations/{id}/title` -- update conversation title.

**Vault endpoints:**
- `POST /api/ai/vault/sync` -- trigger manual sync. Runs VaultSyncService.sync_vault() in background thread. Returns immediately with status "sync_started".
- `GET /api/ai/vault/status` -- returns VaultSyncStatus (file_count, last_sync_at, status).
- `GET /api/ai/vault/files` -- list indexed vault files (file_path, content_hash, last_synced_at).

**Settings endpoints:**
- `GET /api/ai/settings` -- get or create default JojiAISettings for current user. Never expose encrypted token.
- `PUT /api/ai/settings` -- update settings. Encrypt github_token before storing.
